Project Manager Audit

Contract Compliance Report

Evaluated against docs/contracts/V1_Networking_Decisions.md (v3 FROZEN)

This report evaluates the current Python asyncio implementation of the Hyperloop board networking stack at ~/special-lamp. The core implementation shows clean async design patterns, robust protocol handling, and maintains fully passing test coverage (88/88 tests passing). However, several critical gaps must be addressed by the development agent to reach 100% contract compliance.

2. Perfect Implementations (What is Done Well)

  • Orthogonal State Axes (2.1, 2.2): state.py cleanly isolates connection states from the global safety/E-stop axis.
  • Atomic Pop-Wins (1.8): The pop_pending helper in protocol.py uses an atomic lookup-and-pop, guaranteeing that timeout handlers and reader streams never trigger double resolution on the same future.
  • Write Serialization (1.19): All outgoing writes to a board—including normal commands and out-of-band E-stops—serialize under a shared lock (StreamBoardWriter), ensuring newline-JSON framing is never interleaved or corrupted.
  • Unix Socket Lifecycle (0): local_socket.py runs a startup liveness check to prevent dual-controller contention, unlinking dead sockets safely.

3. Critical Gaps Identified (Requires Action)

! Reconnect Jitter & Backoff Decision 1.4

Contract Requirement: Reconnect backoff must be exponential with full jitter: Base 500 ms, factor 2, cap 5 s. Retries are infinite.

Current Implementation: In board_connection.py:L98, the reconnect loop sleeps for a constant delay (self.reconnect_delay_s = 0.05). There is no exponential backoff or jitter implemented.

! Board Liveness Watchdog Decision 1.6

Contract Requirement: Telemetry doubles as the liveness signal. The controller must transition a board to FAULTED after ~5 missed telemetry frames (~250 ms with no inbound message).

Current Implementation: While board_connection.py:L183 stamps last_seen on incoming telemetry, there is no background task running to monitor this timestamp and trigger a disconnect when a board goes silent.

! Local Socket Event Broadcast Loop Decision 1.13, 1.14

Contract Requirement: Local socket clients (GUI) must receive unsolicited event messages (e.g. estop_triggered, state changes).

Current Implementation: LocalUnixSocketServer implements broadcast_event, but nothing in the controller calls it. Controller-observed events only enter the Redis queue; local Unix socket clients never receive E-stop updates or state snapshots.

! Missing orphaned_response Metrics Decision 1.12, 4

Contract Requirement: Client disconnects must drop response delivery and increment the orphaned_response counter.

Current Implementation: deliver_client_response is defined in interfaces.py but is unused. The ControllerCounters class in controller.py is missing the orphaned_response field entirely.

4. Proposed Backlog for Next Agent (Codex)

  1. Implement Exponential Backoff Jitter: Add jittered backoff calculations in board_connection.py, scaling delay from 500 ms up to 5.0 s.
  2. Add Liveness Watchdog: Run a lightweight periodic check on (loop.time() - board.last_seen) > 0.25 in the board connection manager.
  3. Bridge Event Loop: Register a callback handler from LocalUnixSocketServer inside ControllerCore to wire telemetry/state updates directly to client sockets.
  4. Update Metrics: Add `orphaned_response` counter to controller metrics and use deliver_client_response to track dropped client writes.
