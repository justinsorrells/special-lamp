Hyperloop Python Codebase Reference

This study-grade reference gives one tab to every tracked Python source file in the repository. It is companion material, not a protocol extension.

Authority

The frozen controller/networking contract docs/contracts/V1_Networking_Decisions.md remains authoritative, and docs/contracts/Board_Developer_Guide.md governs firmware-facing behavior. If this companion reference disagrees with either, the frozen contracts win.

How To Study The Stack

  • Start with protocol.py, state.py, and interfaces.py; these are the shared contract primitives.
  • Read controller.py before transport files. It owns dispatch, pending resolution, e-stop gating, and shutdown behavior.
  • Read board_connection.py and local_socket.py as adapters around the controller core: TCP boards on one side, full-duplex local Unix clients on the other.
  • Keep the topology in view: local client -> Unix socket -> asyncio controller -> persistent TCP -> board. Redis remains observability/read-replica only.

protocol.py

Layer: shared primitive506 linesRead first

Defines the newline-delimited JSON contract primitives used by every runtime boundary. It deliberately contains no socket, Redis, GUI, or firmware integration, which keeps validation and serialization independent from routing decisions.

Key Symbols

  • MessageType, TerminalStatus, and ErrorCode are closed contract vocabularies.
  • parse_line(), parse_message(), and serialize_message() enforce framing and line limits.
  • build_board_command() rewrites client commands onto the board-facing hop with controller-owned board_seq.
  • build_client_response() restores the client seq and exposes result.board_seq.
  • pop_pending() is the shared pop-wins helper for late, duplicate, timeout, and disconnect races.

Study Notes

  • Client seq and controller board_seq are intentionally separate sequence spaces.
  • controller_ts is a monotonic round-trip token, not wall-clock time.
  • blocked_by_estop defaults to true through command_blocked_by_estop().
  • Malformed input becomes structured errors; callers decide whether to reply, disconnect, or log.

state.py

Layer: shared primitive319 linesState shapes

Holds the in-memory controller state and Redis mirror schemas without performing I/O. This is the best place to understand the two orthogonal axes: per-board connection state and global/per-board safety state.

Key Symbols

  • BoardConnState is the frozen connection axis: disconnected, connecting, connected, registered, faulted.
  • SystemState.estop_active is the global latch cleared only by operator reset.
  • BoardState.estop_ack tracks per-board convergence and is not a dispatch gate.
  • PendingCommand stores board/client sequence mapping plus timeout budgets.
  • BoardStateRecord and SystemStateRecord describe Redis mirror payloads.

Study Notes

  • PendingCommand.__post_init__() enforces the 10 s execution-timeout ceiling.
  • Latency and telemetry rate observations use controller-side measurements.
  • ControllerState.refresh_connected_count() derives connected count from registered board states.
  • Redis records expose current values; they are not authority objects.

interfaces.py

Layer: shared primitive101 linesInjection seams

Defines the protocol-style handoff boundaries between controller components. These seams keep the command path decoupled from concrete sockets, Redis clients, and local-client implementations.

Key Symbols

  • ClientReplyHandle centralizes response/event delivery to local clients.
  • BoardDownHandler is the connection-manager-to-dispatcher drain hook.
  • BoardWriterHandle exposes the serialized per-board write boundary.
  • SerializedBoardWriter wraps a byte-writing callback with newline-JSON serialization and an asyncio.Lock.
  • send_estop() routes out-of-band e-stop through the same writer handle as normal commands.

Study Notes

  • send_estop() bypasses command FIFO and in-flight ownership, but not socket-byte serialization.
  • deliver_client_response() is where orphaned local-client responses are counted.
  • This module should stay free of concrete TCP, Unix socket, Redis, and GUI imports.

controller.py

Layer: controller core1353 linesDispatch owner

Implements the in-memory asyncio controller core. It owns command routing, per-board FIFO behavior, pending resolution, e-stop latching/gating, board-down drains, schema discovery for local clients, lifecycle counters, and graceful shutdown.

Key Symbols

  • ControllerCore is the central dispatcher and state owner.
  • register_board() validates schema protocol version, stores blocked-by-estop metadata, and moves a board to REGISTERED.
  • handle_client_command() routes controller-local commands or board commands.
  • handle_board_response() uses pop-wins to resolve pending board responses.
  • handle_estop_triggered(), handle_estop_reset(), and shutdown() implement safety and lifecycle flows.

Study Notes

  • Only one command can be in flight per board; additional commands enter bounded FIFOs or fail with BOARD_BUSY.
  • Queued commands have a residency cap; execution timeout starts only after board write.
  • In-flight commands are not synthetically failed on e-stop because they may already have physical effects.
  • Observability is injected. The command path must not depend on Redis availability.

board_connection.py

Layer: board TCP adapter521 linesPersistent board transport

Maintains controller-to-board TCP connections. Boards are the TCP servers; this module connects to configured endpoints, requires schema as the first message, forwards unsolicited telemetry/events, and reports disconnects back to the controller core.

Key Symbols

  • BoardEndpoint identifies one statically configured board server.
  • ReconnectBackoff implements exponential full-jitter reconnect delay.
  • StreamBoardWriter serializes command/e-stop writes to one board stream.
  • BoardTCPConnection owns connect, registration, read loop, heartbeat, liveness, and reconnect.
  • HeartbeatConfig and LivenessConfig make optional RX insurance and telemetry liveness explicit.

Study Notes

  • The controller accepts board inbound lines up to 8 KiB; controller outbound commands must fit the board's 1 KiB limit.
  • Registration fails if the first message is not a schema or the board ID/version is wrong.
  • event: estop_ack is unsolicited and flips board.estop_ack through the controller.
  • On read failure or disconnect, board_down() lets the controller fail pending/FIFO entries with BOARD_UNAVAILABLE.

local_socket.py

Layer: local Unix socket adapter404 linesFull-duplex clients

Implements the local client boundary over Unix sockets. Local clients send newline-JSON requests and must continuously read both request-correlated responses and unsolicited controller events.

Key Symbols

  • LocalClientConnection owns one reader/writer pair and one bounded outbound queue.
  • writer_loop() serializes outbound responses/events without blocking the server fan-out.
  • _evict_oldest_noncritical() preserves order while dropping non-critical events under pressure.
  • LocalUnixSocketServer owns socket lifecycle, client registration, request dispatch, and broadcast.
  • _is_critical_event() keeps e-stop/safety events and responses from being silently dropped.

Study Notes

  • Per-client outbound queues isolate slow clients from the command path and other clients.
  • Non-critical events are drop-oldest; critical saturation disconnects the client.
  • The socket path is protected against stale files and live-controller conflicts.
  • This module never talks directly to boards.

observability.py

Layer: observability322 linesRedis side channel

Contains the bounded observability queue, best-effort Redis writer, and serializers for board state, system state, telemetry, controller events, and command lifecycle records.

Key Symbols

  • ObservabilityQueue provides non-awaiting drop-oldest enqueue behavior.
  • ObservabilityCounters tracks dropped observability records and Redis failures.
  • RedisTelemetryWorker drains records and writes hashes, pub/sub deltas, and capped streams.
  • serialize_command_lifecycle() validates terminal statuses before telemetry records are emitted.
  • _redis_value() normalizes hash values to strings.

Study Notes

  • Redis is a read-replica/logging target only; failures are counted and logged, not raised into dispatch.
  • The observability overflow policy is intentionally the opposite of command FIFO behavior.
  • redis=None is a supported disabled mode for tests and local operation.
  • Serialization preserves connection state and safety state as separate fields.

runtime.py

Layer: process composition466 linesStartup/shutdown

Composes the controller core, board TCP connections, local Unix socket server, and optional observability worker into a process runtime. It also loads TOML configuration and installs signal handlers.

Key Symbols

  • RuntimeBoardConfig and RuntimeConfig describe process configuration.
  • ControllerRuntime owns lifecycle ordering for components.
  • load_runtime_config() parses TOML into typed config objects.
  • create_runtime() wires concrete controller, board, local socket, and observability instances.
  • install_signal_handlers() schedules shutdown from SIGINT/SIGTERM.

Study Notes

  • Startup waits for components that must be ready, but reconnecting board tasks keep retrying forever.
  • Shutdown is idempotent and delegates command draining to the controller core.
  • Redis construction is optional and isolated from command-path code.
  • Configuration helpers validate positive numeric values and typed mappings up front.

demos/client/client.py

Layer: demo client207 linesUnix socket CLI

Provides a command-line local client that connects to the controller Unix socket, sends one command or watches events, and prints JSON responses. It demonstrates the expected full-duplex local-client behavior.

Key Symbols

  • UnixSocketDemoClient wraps connect, send, receive, and close operations.
  • run_once() sends a single command and returns the matching response.
  • watch_events() continuously prints unsolicited events/responses from the socket.
  • _parse_args() and _coerce_value() turn CLI args into JSON command args.

Study Notes

  • The demo goes through the Unix socket and never contacts board TCP endpoints.
  • It uses compact protocol messages so examples stay close to the real client boundary.
  • Watching events is not optional for production clients; e-stop and state events can arrive without request correlation.

demos/server/server.py

Layer: demo board354 linesMock TCP board

Implements a mock board TCP server for local testing and demonstrations. It pushes schema on connect, sends telemetry, handles registered commands, and emits e-stop acknowledgements after applying local mock safe state.

Key Symbols

demos/webapp_dashboard.py

Layer: dashboard demo722 linesFastAPI dashboard

Implements a richer web dashboard demo around the local controller socket and optional Redis observability. It exposes board schemas, command forms, state snapshots, event streams, and diagnostics without changing the command path.

Key Symbols

  • ControllerLink manages the Unix socket connection used by the dashboard.
  • build_app() assembles FastAPI routes and background tasks.

Key Symbols
